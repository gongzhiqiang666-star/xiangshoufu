## 关键业务规则总结

### 分润计算规则
```
1. 精度: 向下取整到分
2. 时机: 实时计算（落库同时加入队列）
3. 层级: 无限级，直至没有利润为止
4. 边界: 左闭右闭，如[0,180]第180天算第一阶段
```

### 钱包体系规则
```
1. 返现时效: 实时返现到系统钱包
2. 沉淀钱包: 可使用所有下级余额总和的N%
3. 多钱包扣款: 按余额从多到少扣
4. 贷款利息: 线下协议，系统不处理
5. 提现限额: 按通道+钱包类型分别设置
```

### 代理商体系规则
```
1. 层级深度: 无限级
2. 数据可见: 代理商只能看直属下级，管理员可看所有
3. 流量费统计: 按机具独立计算
```

### 终端管理规则
```
1. 跨级下发: PC端支持跨级，APP端仅限直属下级
2. 货款代扣链: 跨级下发时系统自动按层级生成A→B→C的代扣链
3. 费率同步: 修改后实时同步通道，失败则修改失败
```

### 代扣管理规则
```
1. 伙伴代扣: 任意两个代理商之间都可发起（不限层级关系）
2. 扣款时间: 每日固定时间检查余额并扣款
3. 扣款顺序: 多钱包时按余额从多到少扣
```

### 系统管理规则
```
1. 调账审批: 单级审批，暂无金额上限
2. 批量审核: 部分失败时，成功的通过，失败的标记待处理
3. 模板删除: 有代理商使用时禁止删除
```
